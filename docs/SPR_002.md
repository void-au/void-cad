# SPR-002 — Toolbar

**Sprint Goal:** Deliver a functional toolbar integrated into the application window, providing quick access to viewport controls and laying the groundwork for future tool modes.

**Milestone:** Toolbar  
**Status:** Not Started

---

## Background

With the 3D viewport established, the next step is giving users a proper toolbar. This provides a visual home for mode switching (e.g. select, sketch) and viewport utilities (e.g. reset camera, toggle wireframe). The toolbar should integrate cleanly into the GTK layout alongside the existing `GtkOverlay` viewport.

---

## Tasks

### 1. Layout Restructure
- [ ] Replace the current top-level `GtkOverlay` with a `GtkBox` (vertical orientation) as the window child
- [ ] Place the toolbar widget at the top of the box
- [ ] Place the `GtkOverlay` (GL viewport + FPS label) below, set to expand and fill

### 2. Toolbar Widget
- [ ] Create a `GtkHeaderBar` or `GtkBox`-based toolbar at the top of the window
- [ ] Add a separator to visually divide toolbar groups
- [ ] Style consistently with the application's dark aesthetic

### 3. Viewport Controls Group
- [ ] Add a **Reset Camera** button — resets orbit camera to default position/distance
- [ ] Add a **Wireframe / Solid** toggle button — switches rendering mode (wireframe vs filled)

### 4. Tool Mode Group (placeholder)
- [ ] Add a **Select** mode toggle button (icon + label, inactive for now)
- [ ] Add a **Sketch** mode toggle button (icon + label, inactive for now)
- [ ] Buttons are mutually exclusive (use `GtkToggleButton` group or radio behaviour)

### 5. Camera Reset Implementation
- [ ] Add `reset()` method to `OrbitCamera` that restores default azimuth, elevation, distance, and target
- [ ] Wire the Reset Camera button to call `camera.reset()` and queue a redraw

### 6. Wireframe Toggle Implementation
- [ ] Add a `bool m_wireframe` flag to `Renderer`
- [ ] In `Renderer::draw()`, call `glPolygonMode(GL_FRONT_AND_BACK, m_wireframe ? GL_LINE : GL_FILL)` before draw calls
- [ ] Add a `set_wireframe(bool)` method to `Renderer`
- [ ] Wire the toggle button signal to `renderer->set_wireframe(active)`

### 7. Manual Testing
- [ ] Click Reset Camera and confirm the view returns to the default position
- [ ] Toggle wireframe on/off and confirm the rendering mode changes correctly
- [ ] Confirm toolbar layout does not distort on window resize
- [ ] Confirm placeholder tool buttons are visible but do not crash when clicked

## Technical Notes

- **Toolbar widget:** Prefer `GtkBox` with `GtkButton` / `GtkToggleButton` for full control over layout; `GtkHeaderBar` is an option if title-bar integration is desired
- **Radio behaviour:** Use `gtk_toggle_button_set_group` to link Select / Sketch buttons so only one can be active at a time
- **Wireframe mode:** `glPolygonMode` is a global GL state call — ensure it is reset after draw calls if other geometry (e.g. overlay gizmos) should remain unaffected
- **Icons:** Use named icons from the system theme (`view-grid-symbolic`, `draw-rectangle-symbolic`, etc.) via `gtk_button_new_from_icon_name` where available; fall back to text labels
