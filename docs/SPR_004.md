# SPR-004 — Sketch Mode Foundations

**Sprint Goal:** Deliver an initial sketch mode that lets users enter a 2D drawing context on a plane, create basic sketch geometry, and clearly distinguish sketch interaction from normal 3D navigation.

**Milestone:** Sketch Mode  
**Status:** In Progress

---

## Background

With face and edge selection in place, the next major step is sketching. Sketch mode gives users a controlled 2D workspace anchored to model geometry or a default construction plane. This sprint focuses on the foundations: entering sketch mode, defining the active sketch plane, drawing simple entities, and presenting clear visual feedback so the workflow feels intentional and reliable.

---

## Tasks

### 1. Sketch Mode State
- [x] Add explicit sketch session state in application/runtime data
- [x] Track whether sketch mode is active and which plane is currently being edited
- [x] Prevent conflicting selection/navigation behaviors while sketch mode is active

### 2. Sketch Plane Definition
- [x] Allow sketch mode to start from a selected face when available
- [x] Fall back to a default construction plane when no face is selected
- [x] Derive a stable local 2D plane basis (`origin`, `x_axis`, `y_axis`, `normal`) for sketch coordinates
- [x] Keep orbit camera and pivot behavior compatible with the active sketch plane

### 3. Sketch View Presentation
- [x] Render a visible sketch plane/grid overlay
- [x] Visually indicate the active sketch plane bounds/orientation
- [x] Dim or de-emphasize non-sketch geometry while editing a sketch
- [x] Show a clear status indicator that the user is in sketch mode

### 4. Coordinate Conversion
- [x] Convert mouse position to sketch-plane coordinates
- [x] Support ray-plane hit testing for cursor placement on the active sketch plane
- [x] Keep cursor placement stable across zoom/orbit/pan operations

### 5. Basic Sketch Entities
- [x] Add a lightweight data model for sketch entities
- [x] Support creation of line segments as the first sketch primitive
- [x] Support line preview while positioning the next point
- [x] Store sketch entities relative to the local sketch plane

### 6. Sketch Input Workflow
- [x] On first click, start a line at the current sketch-plane cursor position
- [x] On second click, commit the line segment
- [x] Allow repeated clicks to chain connected line segments
- [x] Support canceling the current in-progress entity with `Esc`
- [x] Provide a clean way to exit sketch mode and return to normal viewport interaction

### 7. Snapping & Precision Basics
- [x] Add optional grid snapping on the active sketch plane
- [x] Snap line endpoints to existing sketch vertices when nearby
- [x] Keep snapping rules simple and deterministic for the first iteration

### 8. Visual Feedback
- [x] Draw committed sketch entities in a clear high-contrast style
- [x] Draw in-progress geometry distinctly from committed geometry
- [x] Highlight current sketch cursor position and snap target
- [x] Ensure sketch overlays remain legible at different zoom levels

### 9. Manual Testing
- [ ] Enter sketch mode with and without a selected face
- [ ] Confirm the sketch plane/grid appears correctly in both cases
- [ ] Draw single and chained line segments on the active plane
- [ ] Cancel an in-progress line and confirm no invalid geometry remains
- [ ] Exit sketch mode and confirm normal selection/navigation resumes cleanly

---

## Technical Notes

- **Plane basis:** Sketch data should be stored in local 2D coordinates and transformed into world space for rendering
- **Source plane:** Prefer selected face plane when available; otherwise use a default canonical plane such as XY
- **Interaction isolation:** Sketch mode should temporarily own left-click drawing behavior rather than normal selection behavior
- **Rendering:** Draw sketch overlays in a dedicated pass so they remain readable above the shaded model
- **Extensibility:** Keep the entity model simple but structured to allow future arcs, circles, dimensions, and constraints
