# SPR-001 — 3D Viewer (Pan, Zoom, Rotate)

**Sprint Goal:** Deliver a functional 3D viewport with interactive camera controls embedded in a native OpenGL application window.

**Milestone:** 3D Viewer (Pan, Zoom, Rotate)  
**Status:** Complete

---

## Background

The 3D viewer is the foundation of Void-CAD. All future features (sketch mode, face selection, toolbars) depend on a reliable, interactive viewport. This sprint establishes the rendering pipeline and camera system.

---

## Tasks

### 1. Project Setup
- [x] Write a `build.sh` script that compiles all sources and links the executable
- [x] Write a `deps.sh` script that checks for / installs required system libraries (GLFW, Epoxy, GLM, OpenGL)
- [x] Set up `src/`, `include/`, `docs/` directory structure
- [x] Confirm a blank application window builds and runs via `./build.sh`

### 2. Window + OpenGL Integration
- [x] Create a native OpenGL viewport as the main view
- [x] Set up an OpenGL context (version 3.3+ core profile)
- [x] Connect render, resize, and input callbacks
- [x] Clear the framebuffer with a solid background colour on each frame

### 3. Basic 3D Rendering Pipeline
- [x] Write a minimal vertex + fragment shader pair (GLSL)
- [x] Set up VAO/VBO for a reference mesh (unit cube or XYZ axis gizmo)
- [x] Implement a `Renderer` class responsible for draw calls
- [x] Apply model/view/projection (MVP) matrices via GLM

### 4. Camera System
- [x] Implement an `OrbitCamera` class with the following state:
  - `target` — the point in world space being looked at
  - `distance` — radius from target
  - `azimuth` / `elevation` — spherical angles
- [x] Derive the view matrix from spherical coordinates each frame

### 5. Mouse Input — Rotate
- [x] Connect mouse motion callbacks for drag events
- [x] On left-button drag: update azimuth and elevation
- [x] Clamp elevation to avoid gimbal flip (e.g. −89° to +89°)

### 6. Mouse Input — Pan
- [x] On middle-button drag: translate the camera `target` in the view plane
- [x] Scale pan speed relative to `distance` so it feels consistent at any zoom level

### 7. Mouse Input — Zoom
- [x] Connect scroll-wheel callbacks
- [x] On scroll: adjust `distance`, clamped to a sensible min/max range
- [ ] Optional: support trackpad pinch-to-zoom

### 8. Viewport Resize Handling
- [x] On `resize` signal: update the projection matrix aspect ratio
- [x] Ensure no distortion when the window is resized

### 9. Manual Testing
- [x] Rotate around a reference object from multiple angles
- [x] Pan the view so the object moves to different screen positions
- [x] Zoom in until near-clip and out to verify distance clamping
- [x] Resize the window and confirm no visual distortion

| # | Criterion |

| 2 | A visible 3D reference object (cube or axis gizmo) is rendered |
| 3 | Left-click drag rotates the view around the target point |

---

## Technical Notes

- **Rendering:** OpenGL 3.3 core profile via GLFW + Epoxy
- **Math:** GLM for vectors, matrices, and projection utilities
- **Camera model:** Orbit/turntable (not free-look) — appropriate for CAD
- **Shader management:** Load GLSL from embedded strings initially; move to files in a later sprint if needed
- **Build system:** Custom bash scripts (`build.sh`, `deps.sh`); use `pkg-config` for compiler/linker flag discovery
